# Active Learning и (опционально) мультимодальные данные

В **типичном end-to-end пайплайне** после разметки идёт шаг **`ActiveLearningAgent`**: отбор информативных примеров из пула, сравнение с **random baseline**, график и метрики. Если пользователь просит **объединить несколько модальностей** (text / image / audio) в согласованный датасет — используй отдельный контракт **`MultimodalAgent`**; не смешивай оба в одном классе без явной просьбы.

## Формат skill

Инструкция для агента; стандарт — [Agent Skills](https://agentskills.io/), примеры — [anthropics/skills](https://github.com/anthropics/skills).

### Папка `scripts/`

`al_cycle_template.py` — каркас AL (fit → query → метки из pool → eval) → `agents/al_agent.py`. `multimodal_align_template.py` — merge таблиц по ключу → `agents/multimodal_agent.py`. Второй нужен только при задачах на мультимодальность.

## Порядок действий

1. Уточни у промпта: нужен ли **только AL**, или ещё **подготовка мультимодального кадра** (без AL или до него).
2. **Интерактивный режим / мастер-пайплайн без autopilot:** перед долгим `run_cycle` согласуй стратегию **`query`** (entropy / margin; сравнение с **random**), при необходимости `n_iterations`, `batch_size`, размер стартового размеченного множества.
3. **AL:** `run_cycle` для основной стратегии и отдельный прогон для **random** → один график (ось X: число размеченных, ось Y: метрика) → краткий текстовый вывод (например, сколько примеров экономится при сопоставимом качестве).
4. **Мультимодальность (если запрошено):** `load_modality` → `align` по документированному ключу (inner/outer и несматченные строки — явно) → `describe` → `export`.

## Ограничения

- Для entropy нужен `predict_proba`; иначе — задокументированный fallback.
- Не теряй несматченные ключи молча: отрази в EDA / отчёте.

## Когда спросить пользователя

- Не ясна стратегия **query** и параметры цикла (и не включён autopilot).
- Не ясно, нужен ли шаг мультимодального merge и какая политика join.
- Выбор **macro vs binary F1** критичен для интерпретации — и не задан в промпте.

Не распрашивай про оформление графиков — разумные подписи осей и легенда по умолчанию.

## Связь с мастер-пайплайном (`data-project-pipeline`)

Оркестратор по умолчанию вызывает **AL** после разметки/HITL. Подготовка мультимодальных таблиц — **по запросу** в сценарии пользователя (порядок и имена шагов зафиксируй в README проекта).

## ActiveLearningAgent — контракт

| Метод | Назначение |
|--------|------------|
| `fit(labeled_df)` | Базовая модель на текущих размеченных данных |
| `query(pool, strategy)` | Индексы строк пула для следующей итерации |
| `evaluate(...)` | Метрики (как минимум accuracy и/или F1 — что зафиксировано в README) |
| `run_cycle(...)` | Полный цикл; возвращает `history` (итерации, n_labeled, метрики) |
| `report(history)` | Сохранение кривой, например `learning_curve.png` |

Стратегии `query`: как минимум **`entropy`**, **`margin`**, **`random`** (random — baseline на том же графике, что и основная стратегия).

```python
from agents.al_agent import ActiveLearningAgent

agent = ActiveLearningAgent(random_state=42)
history = agent.run_cycle(
    labeled_df=df_seed,
    pool_df=df_pool,
    strategy="entropy",
    n_iterations=5,
    batch_size=20,
)
# Отдельно — прогон со strategy="random" для сравнения на одном plot
```

Имена колонок признаков и метки — задокументировать. Симуляция доразметки (метки из ground truth пула) должна быть описана, чтобы цикл был воспроизводим.

## MultimodalAgent — контракт (если нужен)

| Метод | Назначение |
|--------|------------|
| `load_modality(path, type)` | Одна модальность → структура / DataFrame |
| `align(modalities, key)` | Слияние по ключу |
| `describe(aligned_df)` | EDA, в т.ч. покрытие и несматчи |
| `export(aligned_df, format)` | csv / parquet / и т.д. |

Пример смысла: `align({"text": t, "image": m}, key="sample_id")` → единый кадр + отчёт.
